Work in Progress

The Problem

Traditionally, functions are fragmented across CLIs, REST APIs, and various programming languages—each defining inputs and outputs differently. This creates friction and requires constant translation.

The Solution

fnforge introduces a unified approach where every function must explicitly define its Input Type (In) and Output Type (Out), supporting both unary and bidirectional communication without being tied to a specific transport layer.

// The fnforge way
export interface ForgeFunction<In, Out> {
  (input: In | AsyncIterable<In>): Promise<Out> | AsyncIterable<Out>;
}
